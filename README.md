## What this is

This repository documents the integrated modelling framework of **Research Area C3** of the Future Forests Excellence Cluster. It is a registry and coordination tool that describes the modelling groups contributing to C3, what each group does, what data they need and produce, and how they connect to each other.

The repository will be updated as groups develop their projects, connections are confirmed, and new information becomes available.

### `build_registry.py`
Python script that reads `models.yml` and generates `registry.html`.

**Output:** `registry.html` — the full registry page, deployed via GitHub Pages.

**What it does:**
- Groups entries into four thematic clusters (C1/C2, Climate & Hydrology, Forest Ecology & Structure, Socio-Economic)
- Renders a card for each group showing topic, PI, methods, description, inputs, and outputs
- Generates filter buttons for each cluster

---

### `registry.html`
The generated HTML registry. **Do not edit this file directly** — it is overwritten every time `build_registry.py` runs. Edit `models.yml` instead and rebuild.

**Structure:** 16 groups across 4 clusters:

| Cluster | Groups |
|---|---|
| C1 & C2 — Conceptual Framework & Scenarios | C1 SES Framework, C2 Scenarios |
| Climate & Hydrology | Jung/Schindler, Stahl, Christen, Weiler (GIS), Orth |
| Forest Ecology & Structure | Seifert, Kattenborn, iLAND, RoGeR, Stritih |
| Socio-Economic | Hanewinkel, Pauliuk, Schanz, Brown |

Each card shows:
- **Topic** — primary heading (from workshop Excel)
- **PI name** and model name
- **Methods** (from workshop Excel)
- **Short description** (from pitch presentations and proposal)
- **Inputs / Outputs** (from pitch presentations and proposal only)

### `FuFo_Interactive.html`
Standalone interactive connection map. Shows all 16 groups as clickable nodes arranged in the four cluster bands, with arrows representing data flows and lateral connections between groups.

**How to read it:**
- Click a node to highlight its direct connections
- Node title = research topic (from Excel)
- Node subtitle = PI name
- Arrow colours: green = data, orange = bidirectional coupling, blue = validation, purple = conceptual, grey = lateral collaboration
- Use the filter buttons to show one cluster at a time


**Connection types in use:**

| Type | Meaning |
|---|---|
| `data` | One group produces data that another uses as direct input |
| `coupling` | Bidirectional coupling (iLAND ↔ RoGeR) |
| `validation` | Output used for validation/calibration, not direct model input |
| `conceptual` | C1 framework feeds into downstream groups conceptually |
| `lateral` | Methodological or thematic collaboration without direct data flow |

---

### `b_survey.html`
A short survey form for social scientists to describe how their work connects to the C3 modelling framework. Three questions:

1. What does your group produce that is relevant for C3 models?
2. Which C3 groups does your work connect to, and how?
3. What do you need from C3 models to do your work?

Responses are submitted by email to the coordinator via FormSubmit. The form is designed to feed into the registry as RA B connections are confirmed.
